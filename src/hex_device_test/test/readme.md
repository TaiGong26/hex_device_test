关于任务：
- 新建一个仓库作为独立项目，直接通过pip安装hex_device_python
- 抽象化控制接口，方便以后重构后快速切换到新的驱动
- 可以同时接入不同ip、不同端口号的多个机械臂
- 统一发布控制指令，多个机械臂统一执行动作
- 拥有异常状态检测，当任意一个机械臂出现断联或者error的时候，锁定所有机械臂在原位不动，但不退出程序，而是等待操作后才退出程序。
- 拥有单独的线程维护，当驱动直接raise的时候，主线程仍然能保持在线，且能够报告是哪个机械臂退出了。
- 拥有一键归位，退出程序的时候所有机械臂回归到固定的初始化位置

子任务：
- 实现基础功能，能够同步控制不同ip不同端口的多个机械臂，同时要求拥有多线程管理，能够检测线程是否退出。但是不要求有异常状态监测
- 增加状态检测功能，要求维护状态机。状态包括：初始化（等待所有机械臂上线并缓慢到达初始位置）、就绪（到达初始化不动，等待指令）、运行中（所有机械臂同步运动）、异常（所有正常的机械臂锁定在当前位置，等待操作）、停止中（所有机械臂回到出事位置，与初始化不同的是，这里不需要等待机械臂上线）、退出（统计并输出表格，包括：运行状态，运行时间，是否有机械臂出错，机械臂各个关节最高温度等）
优化测试轨迹，标准化测试动作
- 进阶：增加绘图开关，可在运行时随时打开数据输出以查看运行曲线
- 进阶：压力测试，查看最高可以同时运行多少个机械臂，进一步优化代码运行效率
